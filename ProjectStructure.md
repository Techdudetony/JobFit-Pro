# Project Structure

This document describes every file and directory in JobFit Pro and its role in the application.

---

## Top-Level

```
JobFit-Pro/
├── app/                    # UI, layout, and application entry point
├── core/                   # Business logic — parsing, processing, exporting
├── services/               # Shared infrastructure — auth, AI, DB, theme
├── tests/                  # Unit tests (in progress)
├── assets/                 # Icons and static resources
├── .env                    # API keys and config (not committed)
├── requirements.txt        # Python dependencies
├── theme_preference.json   # Persisted dark/light preference
└── README.md
```

### `app/main.py`
Application entry point. Creates `QApplication`, initializes `ThemeManager`, handles the auth gate (auto-login or modal), and launches `MainWindow`.

### `app/window_main.py`
Main window controller. Owns all signal wiring, business logic coordination, and state management. Handles resume loading, job fetch, tailoring, export, history saving, ATS analysis, cloud sync on login, and theme toggling. Contains `TailoringWorker` (QThread subclass for async GPT calls).

### `app/components/file_picker.py`
Reusable `QWidget` for file selection. Supports browse dialog, drag-and-drop, placeholder text, and programmatic path setting. Emits `fileSelected(str)` signal.

### `app/components/output_panel.py`
Formatted resume preview panel using `QTextEdit`. Converts plain resume text to styled HTML (section headers, bullet points, typography). Includes a Copy button.

### `app/components/settings_panel.py`
Tailoring options panel (`QGroupBox` with checkboxes). Exposes `get_settings() → SettingsResult`. Used inside the Tailor tab. Enforces mutual exclusivity between page limit options.

### `app/data/tailoring_history.json`
Local JSON store of all tailoring sessions. Each entry contains company, role, job URL, resume URL, local PDF path, ATS result, cover letter, timestamp, and last_updated.

### `app/data/history_resumes/`
Local PDF copies of each tailored resume, saved at the time of tailoring. Filenames are `YYYYMMDD_HHMMSS_<short-uuid>.pdf`.

### `app/styles/app.qss`
Dark theme QSS stylesheet. Defines colors, typography, button variants, group boxes, auth modal, history table, and message box styles.

### `app/styles/app_light.qss`
Light theme QSS stylesheet. Mirrors dark theme structure with a white/slate color palette.

### `app/ui/auth_modal.py`
Sign-in / sign-up dialog. Handles form validation, password strength check, password visibility toggles, Remember Me checkbox, and Enter-key submission. Delegates all Supabase calls to `AuthManager`. Attempts auto-login from saved session on `showEvent`.

### `app/ui/main_window.py`
Pure UI layout definition for the main window. Builds the sidebar, tab container, and all child tab widgets. Contains no business logic.

### `app/ui/main_window_ui.py`
Auto-generated PyQt6 UI class from `main_window.ui`. Do not edit manually — regenerated by `pyuic6` when the `.ui` file changes.

### `app/ui/main_window.ui`
Qt Designer UI file defining the main window widget layout in XML. Source of truth for the generated `main_window_ui.py`.

### `app/ui/dialogs/about_dialog.py`
About dialog showing app version, tech stack, and developer credit.

### `app/ui/dialogs/help_dialog.py`
Help dialog with tabbed content: User Guide, How Tailoring Works, and About.

### `app/ui/dialogs/loading_dialog.py`
Frameless loading dialog with indeterminate progress bar and animated dots. Shown during AI processing.

### `app/state/session_state.py`
In-memory store for the current working session (resume text, job text, tailored text, file paths). Lightweight and UI-agnostic. Supports save/load to `session_state.json`.

### `app/ui/onboarding.py`
First-launch tutorial overlay. 8-step walkthrough with spotlight highlighting and optional tab switching. Version-gated (`ONBOARDING_VERSION = "v2"`) so existing users see the updated tutorial after upgrades. Skippable; re-triggerable from Help menu.

### `app/ui/sidebar_nav.py`
Custom vertical tab navigation widget. Each tab is a `_TabButton` with icon, label, active state, and an optional red badge dot. Badge auto-clears on tab click. Exposes `set_badge(index, visible)`.

### `app/ui/ats_panel.py`
Sliding ATS score drawer. Renders animated score bar, keyword overlap table, missing skills list, tone analysis, and AI detection result. Loads analysis via `KeywordAnalyzer` in a background thread. Emits `analysisReady(int)` signal when complete.

### `app/ui/toast_notification.py`
Frameless `QWidget` anchored to the bottom-right of its parent. Slides up and fades in on show. Auto-dismisses after a configurable duration. Four presets: `success` (green), `warning` (yellow), `error` (red), `info` (blue).

### `app/ui/tailoring_history_window.py`
Legacy history dialog (pre-v2). Still available via Tools menu as a fallback. The primary history UI is now `tab_history.py`.

### `app/ui/tabs/tab_tailor.py`
Main tailoring tab. Contains the job URL input, Fetch/Manual buttons, job description preview, `FilePicker`, resume preview, `SettingsPanel`, Tailor button, `OutputPanel`, and export buttons.

### `app/ui/tabs/tab_history.py`
History browser tab. Displays all tailoring sessions in a table with ATS score, company, role, resume link, cover letter view button, and delete controls. Supports ATS replay and inline editing.

### `app/ui/tabs/tab_settings.py`
Settings tab. Contains API key input, model selector, theme toggle, default tailoring preferences, and the Cloud Sync panel (Push All to Cloud / Pull from Cloud with last-synced timestamp).

### `app/ui/tabs/tab_cover_letter.py`
Cover letter generator tab. Inputs: tone, length, and highlight. Resume source is selected automatically based on available content. Editable output area. Export to DOCX/PDF. Emits `coverLetterGenerated(str, bool)` for history saving.

## `core/`

Business logic layer. No PyQt6 imports. Safe to unit test independently.

```
core/
├── extractor/
│   ├── pdf_parser.py
│   ├── docx_parser.py
│   └── job_parser.py
├── history/
│   ├── history_manager.py
│   └── utils.py
├── processor/
│   ├── tailor_engine.py
│   ├── cleaner.py
│   ├── keyword_matcher.py
│   ├── keyword_analyzer.py
│   ├── ai_detector.py
│   ├── job_meta_extractor.py
│   └── cover_letter_engine.py
├── uploader/
│   └── supabase_uploader.py
├── utils/
│   └── validators.py
└── exporter/
    ├── docx_builder.py
    └── pdf_exporter.py
```

### `core/extractor/pdf_parser.py`
Extracts text from PDF resumes using `pdfplumber`. Handles multi-page documents, skips unreadable/scanned pages, and normalizes whitespace and unicode artifacts.

### `core/extractor/docx_parser.py`
Extracts text from DOCX resumes using `python-docx`. Processes paragraphs and table cells. Normalizes bullet characters and cleans whitespace.

### `core/extractor/job_parser.py`
Fetches and cleans job descriptions from URLs using `requests` and `BeautifulSoup4`. Strips navigation, scripts, cookies notices, and boilerplate. Returns clean plain text suitable for GPT input.

### `core/processor/tailor_engine.py`
GPT-powered resume tailoring engine. Builds a structured prompt combining the user's resume, job description, and optional length constraints (1–2 pages or strict 1 page). Uses `OpenAIClient` for the API call. Temperature defaults to 0.3 for consistent output.

### `core/processor/cleaner.py`
Text normalization utility. Normalizes bullet variants, curly quotes, dashes, invisible unicode characters, tabs, and excess whitespace. Called on resume text before sending to GPT.

### `core/processor/keyword_matcher.py`
Lightweight ATS keyword overlap scorer. Extracts normalized keywords from job description and resume, returns overlap dict, missing keywords dict, and match rate percentage. Uses a stopword list and supports hyphenated words and acronyms.

### `core/processor/keyword_analyzer.py`
AI-powered ATS breakdown using OpenAI. Returns structured JSON with score, matched/missing keywords, tone assessment, and improvement suggestions. Used by `ATSPanel` for the detailed drawer view.

### `core/processor/ai_detector.py`
Detects AI-generated content in a submitted resume. Scores likelihood and returns a warning flag used in `ATSPanel` to alert the user before tailoring.

### `core/processor/job_meta_extractor.py`
Extracts company name and job title from a raw job description. Uses OpenAI for accuracy on messy or non-standard formats. Falls back to regex heuristics if AI call fails.

### `core/processor/cover_letter_engine.py`
GPT-powered cover letter generator. Accepts resume text, job description, tone, length, and optional highlights. Returns a complete, ready-to-use cover letter.

### `core/history/history_manager.py`
Reads and writes `tailoring_history.json`. Provides `load()` and `add_entry()` methods used by the main window controller after each tailoring session.

### `core/history/utils.py`
Helper for extracting company and role from job description text. Used when building history entries.

### `core/uploader/supabase_uploader.py`
Uploads resume files to Supabase Storage under `resumes/users/<user_id>/<uuid>.<ext>`. Returns a signed URL valid for 7 days, or `None` on failure.

### `core/utils/validators.py`
Reusable, UI-agnostic validation functions for file paths, URLs, text content, and tailoring settings.

### `core/exporter/docx_builder.py`
Converts plain resume text to a formatted DOCX file. Detects section headings, bullet points, and normal paragraphs. Applies ATS-friendly styles (Calibri 11pt, no tables/columns). Uses `python-docx`.

### `core/exporter/pdf_exporter.py`
Exports resume text to PDF by building a temporary DOCX via `docx_builder` and converting it using `docx2pdf`. Returns the final PDF path.

## `services/`

Shared infrastructure layer. Initialized once, imported as singletons.

```
services/
├── config.py
├── openai_client.py
├── auth_manager.py
├── supabase_client.py
├── theme_manager.py
└── sync_manager.py
```

### `services/config.py`
Loads `.env` using `python-dotenv`. Exposes `API_KEY` and `MODEL_NAME`. Raises `ValueError` on startup if `OPENAI_API_KEY` is missing.

### `services/openai_client.py`
Thin wrapper around the OpenAI Python SDK. Provides `generate(prompt, temperature, max_tokens) → str`. Handles exceptions and returns empty string on failure.

### `services/auth_manager.py`
Centralized Supabase authentication manager. Handles sign-up, sign-in, sign-out, Remember Me (keyring token storage), session restore, and grace period logic. Exposes a singleton `auth` instance used throughout the app.

### `services/supabase_client.py`
Creates and exports a single shared Supabase `Client` instance. Reads `SUPABASE_URL` and `SUPABASE_ANON_KEY` from environment. Raises `RuntimeError` if either is missing.

### `services/theme_manager.py`
Manages light/dark theme switching. Reads and writes `theme_preference.json`. Applies QSS stylesheets to the `QApplication`. Exposes a singleton `theme_manager` instance initialized in `main.py`.

### `services/sync_manager.py`
Background sync coordinator. Contains `QThread` workers for pushing/pulling history entries and preferences to/from Supabase. Uses upsert with conflict resolution on `(user_id, timestamp)`. Merge strategy: newer `last_updated` wins; `local_pdf` paths are always preserved from local. Exposes a singleton `sync_manager` instance.

## Supabase Tables

### `tailoring_history`
| Column | Type | Notes |
|---|---|---|
| id | uuid | Primary key, auto-generated |
| user_id | uuid | FK to auth.users |
| company | text | |
| role | text | |
| job_url | text | |
| resume_url | text | Signed Supabase Storage URL |
| local_pdf | text | Local file path |
| cover_letter | text | |
| ats_result | jsonb | Full keyword_analyzer output |
| timestamp | text | ISO8601, unique per user |
| last_updated | text | ISO8601 |
| created_at | timestamptz | Auto |

Unique constraint: `(user_id, timestamp)` — enables safe upsert without duplication.

### `user_preferences`
| Column | Type | Notes |
|---|---|---|
| user_id | uuid | Primary key, FK to auth.users |
| theme | text | `"dark"` or `"light"` |
| settings | jsonb | Tailoring checkbox states |
| updated_at | timestamptz | |
